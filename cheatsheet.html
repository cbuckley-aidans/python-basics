Variables

Think of variables like boxes where you can store information (data) like words, numbers, or True/False values. You give the box a name so you can use the information later.

Syntax

variable_name = value

Examples

# Storing text (string)
player_name = "Sonic"
# Storing a whole number (integer)
score = 100
# Storing a number with decimals (float)
time_left = 45.5
# Storing True or False (boolean)
is_fast = True

When to Use

  • To remember information like a player's score or name.
  • To store numbers for calculations.
  • To keep track of whether something is true or false (like if a game is over).
  • To make your code easier to read by giving names to values.

Common Errors

SyntaxError

Using an Invalid Name

Variable names can't start with numbers and can only contain letters, numbers, and underscores.

1st_place = "Mario"  # Starts with a number (WRONG)
player-score = 50    # Uses a hyphen (WRONG)

Fix: Start with a letter or underscore. Use underscores instead of hyphens.

first_place = "Mario" # CORRECT
player_score = 50     # CORRECT
SyntaxError / NameError

Forgetting Quotes for Text

Text data (called strings) needs to be inside single (' ') or double (" ") quotes. Without them, Python thinks it's a variable name.

message = Hello      # Missing quotes (WRONG)

Fix: Put quotes around the text.

message = "Hello"    # CORRECT
message = 'Hello'    # Also CORRECT
NameError

Using a Variable Before Creating It

You need to give a variable a value (assign it) before you can use it.

print(my_score)  # my_score hasn't been created yet (WRONG)
my_score = 100

Fix: Make sure you create the variable with `=` before you try to print it or use it.

my_score = 100
print(my_score)  # CORRECT

print() Function

The `print()` function is how you make your program show messages, numbers, or the values stored in variables on the screen.

Syntax

print(value_to_show)

Examples

# Printing simple text
print("Game Over!")
# Printing a number
print(100)
# Printing the value of a variable
score = 500
print(score)
# Printing multiple things (they appear with spaces in between)
level = 3
print("Level:", level, "Score:", score) # Output: Level: 3 Score: 500

When to Use

  • To show messages to the user (like instructions or results).
  • To check the value of a variable while you're coding (debugging).
  • To display calculations or final scores.
  • To see what your program is doing step-by-step.

Common Errors

SyntaxError

Forgetting Parentheses `()`

`print` is a function, and functions need parentheses `()` after their name, even if there's nothing inside (in Python 3).

print "Hello"  # Missing parentheses (WRONG - Old Python 2 style)

Fix: Always add parentheses after `print`.

print("Hello") # CORRECT
NameError

Forgetting Quotes Around Text

If you want to print exact text, it must be inside single `' '` or double `" "` quotes. Otherwise, Python thinks it's a variable name.

print(Hello)   # Python thinks Hello is a variable (WRONG)

Fix: Add quotes around the text you want to display.

print("Hello") # CORRECT
SyntaxError

Mismatched Quotes

If you start text with a double quote (`"`), you must end it with a double quote. Same for single quotes (`'`).

print("Hello') # Started with " but ended with ' (WRONG)

Fix: Use the same type of quote at the beginning and end.

print("Hello") # CORRECT
print('Hello') # Also CORRECT

input() Function

The `input()` function lets your program pause and wait for the user to type something in. Whatever the user types is then stored (usually in a variable) as text (a string).

Syntax

variable_name = input("Optional message to show the user: ")

Examples

# Ask for the user's name and store it
name = input("What is your name? ")
print("Hello,", name)

# Ask for age (input always gives text, even if it looks like a number!)
age_text = input("How old are you? ")
print("You said you are", age_text)

# If you need the age as a number for math, you must convert it
try:
    age_number = int(age_text) # int() turns text into a whole number
    next_year_age = age_number + 1
    print("Next year you will be", next_year_age)
except ValueError:
    print("That wasn't a valid number for age!")

When to Use

  • To make your program interactive.
  • To get information from the user, like their name, age, or choices.
  • To ask questions in a quiz game.
  • To get settings or options from the user.

Common Errors

TypeError

Trying Math with the Input Directly

`input()` gives you text. You can't do math like addition (`+`) with text and numbers directly.

age = input("How old are you? ") # age is text, e.g., "12"
next_age = age + 1                 # Can't add "12" and 1 (WRONG)

Fix: Convert the input text to a number using `int()` (or `float()`) before doing math.

age_text = input("How old are you? ")
try:
    age_number = int(age_text)      # Convert to number
    next_age = age_number + 1       # Now math works (CORRECT)
    print(next_age)
except ValueError:
    print("Please enter a number for age.")
NameError

Forgetting to Store the Input

`input()` gets the user's text, but if you don't store it in a variable using `=`, the text is lost immediately.

input("What is your name? ") # Gets the name but doesn't save it
print("Hello,", name)       # 'name' variable doesn't exist (WRONG)

Fix: Assign the result of `input()` to a variable.

name = input("What is your name? ") # Store the input in 'name'
print("Hello,", name)              # Now 'name' has the value (CORRECT)
ValueError

User Enters Non-Number When Expecting Number

If you use `int()` or `float()` to convert input, but the user types text like "twelve" or "abc", the conversion will fail and raise a `ValueError`.

age_text = input("Age: ") # User types "ten"
age_number = int(age_text) # Crashes here! (ValueError)

Fix: Use a `try...except ValueError` block to catch the error and handle it gracefully (e.g., ask the user again).

while True:
    age_text = input("Age: ")
    try:
        age_number = int(age_text)
        break # Exit loop if conversion is successful
    except ValueError:
        print("Invalid input. Please enter a number.")
print("Age entered:", age_number)

if / elif / else

These statements let your program make choices. It checks a condition (`True` or `False`). If the `if` is true, its code runs. If not, it checks `elif` conditions. If none are true, the `else` code runs (if present).

Syntax

if condition_is_true:
    # Code to run if the condition is True (must be indented!)
elif another_condition_is_true:
    # Code to run if the first condition was False,
    # but this 'elif' condition is True (indented!)
else:
    # Code to run if NONE of the above 'if' or 'elif'
    # conditions were True (indented!)

Examples

score = 75

if score >= 90:
    print("Grade: A")
elif score >= 80: # Only checked if score was NOT >= 90
    print("Grade: B")
elif score >= 70: # Only checked if score was NOT >= 80
    print("Grade: C")
else: # Only runs if score was < 70
    print("Grade: Needs Improvement")
# Output: Grade: C

weather = "cloudy"
if weather == "sunny":
    print("Wear sunglasses!")
elif weather == "rainy":
    print("Bring an umbrella!")
else: # Runs because weather is not "sunny" or "rainy"
    print("Maybe wear a light jacket.")
# Output: Maybe wear a light jacket.

When to Use

  • To run different code based on user input or choices.
  • To check if a player has enough points, lives, or items.
  • To decide what message to show based on a result or status.
  • To control the flow and logic of your program.

Common Errors

SyntaxError

Forgetting the Colon `:`

Every `if`, `elif`, and `else` line must end with a colon `:`.

if score > 50  # Missing colon (WRONG)
    print("You passed!")

Fix: Add a colon `:` at the end of the condition line.

if score > 50: # CORRECT
    print("You passed!")
IndentationError

Incorrect Indentation

The code that should run *inside* the `if`, `elif`, or `else` must be pushed in (indented). If it's not, Python doesn't know it belongs there.

if score > 50:
print("You passed!") # Not indented (WRONG)

Fix: Press Tab or add 4 spaces before the code inside the block.

if score > 50:
    print("You passed!") # CORRECT (Indented)
SyntaxError

Using `=` Instead of `==` for Checking Equality

A single equals sign `=` is for *assigning* a value. A double equals sign `==` is for *checking* if two things are equal in conditions.

user_input = "Admin"
if user_input = "Admin": # Trying to assign, not check (WRONG)
    print("Welcome, Admin!")

Fix: Use `==` when you want to compare if two things are the same inside an `if` or `elif`.

user_input = "Admin"
if user_input == "Admin": # CORRECT (Checks if user_input is equal to "Admin")
    print("Welcome, Admin!")

Booleans & Comparisons

Booleans are `True` or `False`. They result from comparisons (`==`, `>`, `<`, `!=`, etc.) and are used in `if` statements and logical operations (`and`, `or`, `not`).

Comparison Operators

x = 10
y = 5
z = 10

# Equality
print(x == z)  # Is x equal to z?       -> True
print(x == y)  # Is x equal to y?       -> False

# Inequality
print(x != y)  # Is x NOT equal to y?   -> True
print(x != z)  # Is x NOT equal to z?   -> False

# Greater/Less Than
print(x > y)   # Is x greater than y?   -> True
print(x < y)   # Is x less than y?      -> False
print(x >= z)  # Is x greater than or equal to z? -> True
print(y <= 5)  # Is y less than or equal to 5?    -> True

Logical Operators

age = 15
has_permission = False
score = 85

# --- and --- : BOTH sides MUST be True for the result to be True
can_drive = age >= 16 and has_permission
print(f"Can drive? {can_drive}") # Output: Can drive? False (age is >=16 is False)

# --- or --- : AT LEAST ONE side must be True for the result to be True
has_high_score = score > 90 or age < 18
print(f"High score or young? {has_high_score}") # Output: High score or young? True (age < 18 is True)

# --- not --- : Flips True to False, and False to True
is_adult = age >= 18
is_minor = not is_adult
print(f"Is minor? {is_minor}") # Output: Is minor? True (not False is True)

When to Use

  • Inside `if`, `elif` conditions to decide which code path to take.
  • To store the state of something (e.g., `is_game_over = True`).
  • To combine multiple checks before making a decision (e.g., `if is_logged_in and has_admin_rights:`).
  • In `while` loops to decide when to stop looping (more advanced).

Common Errors

SyntaxError

Using `=` Instead of `==` for Comparison

Single `=` assigns a value, double `==` checks if values are equal. This is very common in `if` statements.

mode = "easy"
if mode = "easy": # Assigns "easy" to mode, doesn't check! (WRONG)
    print("Starting easy mode.")

Fix: Use `==` for checking equality in conditions.

mode = "easy"
if mode == "easy": # Checks if mode is equal to "easy" (CORRECT)
    print("Starting easy mode.")
NameError

Incorrect Capitalization of `True` / `False`

Python is case-sensitive. The boolean values must be `True` and `False` (starting with a capital letter).

is_raining = true  # Lowercase 't' (WRONG)
if is_raining:
    print("Bring umbrella.")

Fix: Capitalize `True` and `False`.

is_raining = True  # Capital 'T' (CORRECT)
if is_raining:
    print("Bring umbrella.")
Logic Error

Confusing `and` / `or` Logic

Using `and` when `or` is needed, or vice versa, leads to incorrect results. `and` requires *all* parts to be true, `or` requires only *one* part to be true.

# Trying to check if a number is outside the range 0-100
value = -5
if value >= 0 and value <= 100: # Checks if INSIDE the range (WRONG LOGIC for goal)
    print("Value is valid.")
else:
    print("Value is outside the range.") # This will print for -5

Fix: Think carefully. To check if it's *outside* the range, you want to know if it's less than 0 *OR* greater than 100.

value = -5
if value < 0 or value > 100: # Check if below 0 OR above 100 (CORRECT LOGIC)
     print("Value is outside the range.")
else:
     print("Value is valid.")

Random Module

The `random` module provides functions for generating random numbers, choosing random items, and shuffling sequences.

Syntax (Import First!)

import random # You MUST do this first

# --- Common Functions ---
random_number = random.randint(start, end)   # Inclusive integer
random_item = random.choice(list_or_sequence) # Pick one item
random.shuffle(my_list)                      # Shuffle list in-place
random_decimal = random.random()             # Float 0.0 <= x < 1.0

Examples

import random

# Roll a 6-sided die
dice_roll = random.randint(1, 6)
print(f"You rolled: {dice_roll}")

# Pick a random color
colors = ["red", "green", "blue", "yellow"]
chosen_color = random.choice(colors)
print(f"Random color: {chosen_color}")

# Shuffle a list of players
players = ["Alice", "Bob", "Charlie"]
random.shuffle(players) # Shuffles the 'players' list in place
print(f"Shuffled players: {players}")

# Get a random float
random_float = random.random()
print(f"Random float: {random_float}")

When to Use

  • Making games more unpredictable (dice rolls, enemy actions, random events).
  • Choosing random questions in a quiz or random elements for display.
  • Shuffling items in a list (like a deck of cards or player order).
  • Generating random test data or sample values.

Common Errors

NameError

Forgetting to Import `random`

Python doesn't automatically know about the `random` tools. You have to tell it to load them using `import random`.

dice = random.randint(1, 6) # Forgot 'import random' first (WRONG)
print(dice)

Fix: Add `import random` at the very beginning of your Python file.

import random # CORRECT
dice = random.randint(1, 6)
print(dice)
TypeError

Wrong Arguments for `randint`

`random.randint()` needs exactly two numbers: the lowest possible number and the highest possible number (inclusive).

import random
# Trying to get a number up to 6
dice = random.randint(6) # Only gave one number (WRONG)
print(dice)

Fix: Provide both the minimum and maximum values.

import random
# Get a number between 1 and 6
dice = random.randint(1, 6) # Needs start and end values (CORRECT)
print(dice)
TypeError

Using `random.choice` on Something That's Not a Sequence

`random.choice()` needs something it can pick *from*, like a list (`[]`), a tuple (`()`), or a string. It can't pick from a single number.

import random
number = 10
pick = random.choice(number) # Can't choose from a single integer (WRONG)
print(pick)

Fix: Give `random.choice()` a list, tuple, or string.

import random
options = [10, 20, 30] # A list
pick = random.choice(options) # Choose from the list (CORRECT)
print(pick)

name = "PYTHON" # A string
letter = random.choice(name) # Choose a random character (CORRECT)
print(letter)
Logic Error / TypeError

Expecting `random.shuffle` to Return a New List

`random.shuffle()` shuffles the list *in place* (modifies the original) and returns `None`. Assigning its result leads to `None`.

import random
my_list = [1, 2, 3, 4]
shuffled = random.shuffle(my_list) # WRONG - shuffle changes my_list, returns None
print(my_list)  # This WILL be shuffled
print(shuffled) # This will print None

Fix: Don't assign the result. Call `random.shuffle()` on the list, then use the original (now shuffled) list variable.

import random
my_list = [1, 2, 3, 4]
random.shuffle(my_list) # Shuffles my_list directly (CORRECT)
print(my_list) # Prints the now shuffled list, e.g., [3, 1, 4, 2]

Import Statement

The `import` statement brings external code (modules or libraries) into your program, giving it new capabilities.

Syntax (Common Ways)

# Method 1: Import the whole module
import module_name
# Usage: module_name.function()

# Method 2: Import specific things
from module_name import specific_function, specific_variable
# Usage: specific_function()

# Method 3: Import with a nickname (alias)
import module_name as nickname
# Usage: nickname.function()

Examples

# --- Using Method 1 ---
import math
print(math.sqrt(16)) # Output: 4.0
print(math.pi)       # Output: 3.14159...

# --- Using Method 2 ---
from random import randint, choice
dice_roll = randint(1, 6) # No 'random.' prefix needed
colors = ["red", "blue"]
pick = choice(colors)     # No 'random.' prefix needed
print(f"Rolled {dice_roll}, picked {pick}")

# --- Using Method 3 ---
import datetime as dt # Use 'dt' as nickname
now = dt.datetime.now()
print(f"Current time: {now}")

When to Use

  • When you need functions not built directly into Python (like `random.randint`, `math.sqrt`).
  • To use code written by others (external libraries) or code in separate files.
  • To keep code organized using specialized tools from modules.

Common Errors

NameError

Forgetting to Import

If you try to use a function/variable from a module without importing it first, Python won't recognize the name.

root = math.sqrt(25) # Forgot 'import math' (WRONG)
print(root)

Fix: Add the correct `import` statement (e.g., `import math`) at the top of your code.

import math # CORRECT
root = math.sqrt(25)
print(root)
ModuleNotFoundError / AttributeError

Typo in Module or Function Name

Spelling counts! If you misspell the module name (`import maths`) or the function/variable you're using (`math.sqroot`), Python can't find it.

import maths      # Should be 'math' (ModuleNotFoundError)
# OR
import math
root = math.sqroot(9) # Should be 'sqrt' (AttributeError)

Fix: Double-check the spelling of the module and its contents.

import math      # CORRECT module name
root = math.sqrt(9) # CORRECT function name
NameError

Forgetting the Module Name Prefix (When using `import module_name`)

If you use `import module_name`, you MUST put `module_name.` before using anything from that module.

import random
dice = randint(1, 6) # Forgot 'random.' prefix (WRONG)
print(dice)

Fix: Add `module_name.` (e.g., `random.randint(1, 6)`), OR change import to `from random import randint` and use `randint(1, 6)` directly.

import random
dice = random.randint(1, 6) # CORRECT use with prefix
print(dice)
# --- OR ---
from random import randint
dice = randint(1, 6)       # CORRECT use without prefix (after 'from...import')
print(dice)
ImportError / AttributeError

Circular Import

(Advanced) Happens if file A imports file B, and file B imports file A, creating a loop Python can't resolve during import.

# file_a.py
import file_b # A imports B
# ...
# file_b.py
import file_a # B imports A (Potential circular dependency)
# ...

Fix: Reorganize code. Move shared elements to a third file, or import within functions instead of globally if possible.

For Loops

A `for` loop repeats code for each item in a sequence (like a list, range of numbers, or string).

Syntax

for temporary_variable in sequence:
    # Code block to repeat (must be indented!)
    # Runs once for each item, 'temporary_variable' holds current item.

Examples

# --- Looping through numbers using range() ---
# range(5) -> 0, 1, 2, 3, 4
print("Counting 0 to 4:")
for i in range(5):
    print(i)

# range(2, 6) -> 2, 3, 4, 5
print("\nCounting 2 to 5:")
for number in range(2, 6):
    print(number)

# range(0, 10, 2) -> 0, 2, 4, 6, 8 (start, stop, step)
print("\nCounting evens up to 8:")
for count in range(0, 10, 2):
    print(count)

# --- Looping through items in a list ---
print("\nShopping List:")
shopping_list = ["apples", "bread", "milk"]
for item in shopping_list:
    print(f"- Buy {item}")

# --- Looping through characters in a string ---
print("\nLetters in 'Loop':")
word = "Loop"
for letter in word:
    print(letter)

When to Use

  • To do the same action for every item in a list.
  • To run code a specific number of times (using `range()`).
  • To process each character in text individually.
  • Iterating through sequences like lists, tuples, strings, dictionaries.

Common Errors

SyntaxError

Forgetting the Colon `:`

The line starting with `for` must end with a colon `:`.

for i in range(3) # Missing colon (WRONG)
    print(i)

Fix: Add a colon `:` at the end of the `for` line.

for i in range(3): # CORRECT
    print(i)
IndentationError

Incorrect Indentation

The code to be repeated inside the loop must be indented consistently.

pets = ["cat", "dog"]
for pet in pets:
print(pet) # Not indented (WRONG)

Fix: Indent the code inside the loop.

pets = ["cat", "dog"]
for pet in pets:
    print(pet) # CORRECT (Indented)
IndexError

Off-by-One Errors with `range()`

`range(n)` goes from `0` up to `n-1`. Using `range(len(list))` gives valid indices, but `range(len(list) + 1)` or trying index `n` will cause an `IndexError`.

my_list = [10, 20, 30] # Indices are 0, 1, 2
# range(len(my_list) + 1) is range(4) -> 0, 1, 2, 3
for i in range(len(my_list) + 1): # WRONG - i will become 3
    print(my_list[i]) # IndexError when i is 3

Fix: Use `range(len(my_list))`, or preferably, loop directly over items: `for item in my_list:`.

my_list = [10, 20, 30]
# Correct way using range and index
for i in range(len(my_list)): # range(3) -> 0, 1, 2
    print(f"Index {i}: {my_list[i]}")

# Simpler, preferred way
for item in my_list:
    print(f"Item: {item}")
Logic Error / Unexpected Behavior

Modifying a List While Looping Through It

(Advanced) Adding/removing items from a list while iterating over that same list can confuse the loop, leading to skipped items or errors.

numbers = [1, 2, 3, 4, 5]
for number in numbers:
    if number % 2 == 0: # If number is even
        numbers.remove(number) # Removing items while looping (RISKY!)
print(numbers) # Output might be unexpected, e.g., [1, 3, 5]

Fix: Iterate over a *copy* (`numbers[:]`) if modifying the original, or build a *new* list instead.

numbers = [1, 2, 3, 4, 5]
odd_numbers = [] # Create a new list
for number in numbers:
    if number % 2 != 0: # If number is odd
        odd_numbers.append(number) # Add to the new list
print(odd_numbers) # Output: [1, 3, 5] (Safer)

# Alternative using list comprehension (more advanced)
odd_numbers_comp = [num for num in numbers if num % 2 != 0]
print(odd_numbers_comp) # Output: [1, 3, 5]

Functions

Functions package code into reusable blocks. Define them with `def`, give them a name, optionally specify input parameters, and optionally use `return` to send back a result.

Syntax (Defining)

def function_name(parameter1, parameter2, ...):
    # Function Body (Indented!)
    # Code to execute when called.
    # It can use parameter values.
    # ... some actions ...
    return result_value # Optional: Sends value back

Syntax (Calling)

# Call function (no args, no return needed)
function_name()

# Call function with arguments
function_name(argument1, argument2)

# Call function that returns a value
result = function_name(argument1)

Examples

# --- Defining Functions ---
# 1. Simple: No inputs, no return
def show_menu():
    print("1. Start Game")
    print("2. Options")

# 2. With input (parameter): name
def greet(name):
    print(f"Hello, {name}! Welcome.")

# 3. With inputs, returns a value
def calculate_area(length, width):
    area = length * width
    return area

# --- Calling the Functions ---
show_menu() # Runs the print statements

greet("CyberKid") # Passes "CyberKid" as argument for 'name'

room_length = 10
room_width = 5
# Calls calculate_area, return value (50) is stored
square_feet = calculate_area(room_length, room_width)
print(f"Area: {square_feet} sq ft") # Output: Area: 50 sq ft

When to Use

  • For code used multiple times (avoid repetition).
  • To break large programs into smaller, manageable, named parts.
  • To make code easier to test and debug.
  • To create reusable tools.

Common Errors

No Error (Code Doesn't Run)

Defining a Function but Forgetting to Call It

Defining a function with `def` only sets it up. You must call it by name `()` later to execute its code.

def start_engine():
    print("Engine started!")

# "Engine started!" is never printed because start_engine() wasn't called.

Fix: Add a line to call the function: `start_engine()`.

def start_engine():
    print("Engine started!")

print("Preparing...")
start_engine() # CORRECT - Calls the function
print("Ready!")
TypeError

Wrong Number of Arguments When Calling

If a function definition specifies parameters, you must provide the same number of arguments when calling it.

def set_volume(level): # Defined with 1 parameter
    print(f"Volume set to {level}")

set_volume() # Called with 0 args (TypeError - missing argument)
set_volume(10, 20) # Called with 2 args (TypeError - too many arguments)

Fix: Provide the correct number of arguments.

def set_volume(level):
    print(f"Volume set to {level}")

set_volume(11) # CORRECT - Called with 1 argument
Logic Error / TypeError (later)

Trying to Use a Result Without `return`

If a function calculates a value but doesn't `return` it, the value is lost when the function ends. Calling the function yields `None`.

def get_player_score(points):
    score = points * 10
    # Forgot to return score

current_score = get_player_score(5) # current_score becomes None
print(f"Your score is: {current_score}") # Prints: Your score is: None

Fix: Add a `return` statement to send the value back.

def get_player_score(points):
    score = points * 10
    return score # CORRECT - Sends score back

current_score = get_player_score(5)
print(f"Your score is: {current_score}") # Prints: Your score is: 50
NameError

Scope Issue: Variable Only Exists Inside Function

Variables created inside a function (local variables) can't be accessed from outside that function after it finishes.

def create_message():
    message = "Function finished!" # 'message' is local
    print(message)

create_message()
print(message) # Trying to access 'message' outside (NameError)

Fix: `return` the value from the function and store it outside, or use a global variable (carefully).

def create_message():
    local_message = "Function finished!"
    return local_message # Return the value

msg_from_func = create_message() # Store the returned value
print(msg_from_func) # CORRECT - Access the stored return value

Turtle Graphics

The `turtle` module allows drawing shapes and patterns by controlling a virtual "turtle" on a screen.

Syntax (Basic Setup)

import turtle # 1. Import

screen = turtle.Screen() # 2. Create screen (optional but good practice)
screen.bgcolor("lightblue")

pen = turtle.Turtle() # 3. Create turtle
pen.shape("turtle")
pen.color("red")

# 4. Give commands
pen.forward(100)
pen.left(90)
# ... more commands ...

# 5. Keep window open
screen.exitonclick() # Or turtle.done()

Common Turtle Commands (`pen.`)

# --- Movement ---
forward(dist); backward(dist); right(deg); left(deg); goto(x, y); setheading(deg); circle(radius)

# --- Pen Control ---
penup(); pendown(); pensize(width); pencolor("color"); speed(num)

# --- Appearance & Filling ---
shape("name"); fillcolor("color"); begin_fill(); end_fill(); hideturtle(); showturtle()

# --- Screen Commands (`screen.`) ---
bgcolor("color"); title("text"); setup(w, h); exitonclick()

When to Use

  • To visually learn programming concepts (loops, functions).
  • To draw geometric shapes, patterns, fractals.
  • For simple animations or interactive drawings.
  • For fun introductory coding projects.

Common Errors

NameError

Forgetting `import turtle`

Python doesn't know turtle commands unless the module is imported.

t = turtle.Turtle() # Forgot 'import turtle' (NameError)
t.forward(50)

Fix: Add `import turtle` at the top of your file.

import turtle # CORRECT
t = turtle.Turtle()
t.forward(50)
No Error (Window Vanishes)

Window Closes Immediately

The script finishes, closing the graphics window before you see the result.

import turtle
t = turtle.Turtle()
t.circle(50)
# Program ends, window might vanish (MISSING WAIT)

Fix: Add `screen.exitonclick()` (if using `Screen`) or `turtle.done()` at the very end.

import turtle
screen = turtle.Screen() # Create screen
t = turtle.Turtle()
t.circle(50)
screen.exitonclick() # CORRECT - Waits for click
AttributeError / TurtleGraphicsError

Typo in Command or Color Name

Command names (`forward`, `pencolor`) and standard color names ("lightblue") must be spelled exactly correctly (case-sensitive).

import turtle
t = turtle.Turtle()
t.forwad(100) # Misspelled 'forward' (AttributeError)
t.pencolor("liteblue") # Misspelled 'lightblue' (TurtleGraphicsError)

Fix: Double-check spelling against documentation.

import turtle
t = turtle.Turtle()
t.forward(100) # CORRECT
t.pencolor("lightblue") # CORRECT
AttributeError

Mixing up Turtle and Screen Commands

Some commands belong to the turtle object (`pen.forward()`), others to the screen object (`screen.bgcolor()`).

import turtle
t = turtle.Turtle()
t.bgcolor("yellow") # bgcolor belongs to screen (AttributeError)

Fix: Call commands on the correct object. Create a `screen = turtle.Screen()` and use `screen.bgcolor()` etc.

import turtle
screen = turtle.Screen() # Create screen object
t = turtle.Turtle()      # Create turtle object

screen.bgcolor("yellow") # CORRECT - Call bgcolor on the screen
t.forward(100)           # CORRECT - Call forward on the turtle
screen.exitonclick()
