Functions: Reusable Code Blocks

A function is like a mini-program within your program. It's a block of code that performs a specific task, which you can use whenever you need that task done.

1

Function Metaphor

Think of a function like a blender:

Input (Parameters)

Ingredients you put in

Process (Function body)

Blending the ingredients

Output (Return value)

Smoothie you get out

2

Basic Syntax

In Python, we define functions using this pattern:

def function_name(): # Indented block of code

For example:

def say_hello(): # This is the function body print("Hello, World!") # To use (call) the function: say_hello()

The def keyword tells Python you're creating a function. The parentheses () can hold parameters (we'll learn about those later). The colon : and indentation create a code block.

3

Anatomy of a Function

Function Anatomy Step 1 of 3
def greet_user(): print("Welcome to Python!") print("Functions are awesome!") greet_user()

Function Definition Line

  • This is where you create and name your function
  • The def keyword tells Python you're creating a function
  • The function name greet_user should describe what the function does
  • The parentheses () can hold parameters (we'll learn about these later)
  • The colon : marks the start of the function body
def greet_user(): print("Welcome to Python!") print("Functions are awesome!") greet_user()

Function Body

  • The indented block of code that runs when the function is called
  • It can contain multiple lines of code that perform a specific task
  • All lines must be indented (usually 4 spaces)
  • The indentation tells Python which lines belong to the function
  • When the function is called, Python executes these instructions in order
def greet_user(): print("Welcome to Python!") print("Functions are awesome!") greet_user()

Function Call

  • This is how you use (or "call") your function
  • The function name followed by parentheses tells Python to run the code in the function
  • When Python sees this line, it jumps to the function definition, runs the code inside, then returns to this point
  • You can call a function multiple times in your program
Welcome to Python! Functions are awesome!

Making Functions Flexible with Parameters

Parameters are like inputs for your function. They allow you to pass different values to your function, making it more flexible and reusable.

1

Understanding Parameters

A parameter is a variable that receives a value when the function is called:

def greet(name): print("Hello,", name)

Call the function with different arguments:

greet("Alex") # Output: Hello, Alex greet("Jamie") # Output: Hello, Jamie

Here, name is a parameter that accepts different values each time the function is called.

Parameter vs. Argument

Parameter: The variable listed in the function definition def greet(name)

Argument: The value passed when calling the function greet("Alex")

2

Multiple Parameters

Functions can have multiple parameters, separated by commas:

Multiple Parameters Example Step 1 of 3
def calculate_rectangle_area(length, width): area = length * width print("The area of the rectangle is:", area)

This function has two parameters: length and width. It calculates the area by multiplying them together.

def calculate_rectangle_area(length, width): area = length * width print("The area of the rectangle is:", area) calculate_rectangle_area(5, 3)

Here we call the function with arguments 5 and 3. These values are passed to the parameters, and the function calculates: 5 Ã— 3 = 15.

The area of the rectangle is: 15
def calculate_rectangle_area(length, width): area = length * width print("The area of the rectangle is:", area) calculate_rectangle_area(5, 3) calculate_rectangle_area(10, 20)

We call the same function again with different arguments (10 and 20). This shows how functions are reusable - we use the same code with different inputs.

The area of the rectangle is: 15 The area of the rectangle is: 200

Default Parameter Values

Python allows you to set default values for parameters. If no argument is provided when calling the function, the default value is used:

def greet(name, greeting="Hello"): print(greeting + ", " + name + "!") # Call with both parameters greet("Alex", "Hi") # Output: Hi, Alex! # Call with only required parameter, the default greeting is used greet("Jamie") # Output: Hello, Jamie!

Key Points About Parameters

  • Parameters make your functions flexible and reusable
  • Parameters act like variables inside the function
  • You can have multiple parameters separated by commas
  • Default values can make some parameters optional
  • The order of arguments must match the order of parameters (unless you use named arguments)

Common Coding Mistakes

Common Error:

def greet(name): print("Hello, " + name)

Problem: Missing indentation in the function body.

Fix: Indent all lines in the function body.

def greet(name): print("Hello, " + name)

Common Error:

def add_numbers(a, b): return a + b result = add_numbers(5) # Missing second argument

Problem: Not providing all required arguments.

Fix: Provide values for all parameters (or use default values).

def add_numbers(a, b): return a + b result = add_numbers(5, 10) # Provide both arguments

Getting Results from Functions

So far, our functions have done things (like printing messages), but they haven't given us back any values. The return statement allows functions to send values back for use elsewhere in our program.

1

Return Value Basics

The return statement sends a value back to where the function was called:

def add_numbers(a, b): return a + b # Send back the sum # Store the returned value in a variable result = add_numbers(5, 3) # Use the returned value print("The sum is:", result) # Output: The sum is: 8

Why Use Return Values?

  • Lets you use the result elsewhere in your code
  • Makes functions more reusable
  • Allows functions to work together
2

3

Using Return Values

Return values make functions more powerful by letting you combine them:

def calculate_rectangle_area(length, width): return length * width def calculate_volume(length, width, height): # Use the first function to calculate the base area base_area = calculate_rectangle_area(length, width) # Then calculate the volume return base_area * height # Call the volume function box_volume = calculate_volume(5, 3, 2) print("The volume is:", box_volume) # Output: The volume is: 30

Notice how the second function uses the first function's return value. This makes your code more modular and easier to maintain.

Important Notes About Return Values

1

EARLY RETURNS

A return statement immediately exits the function, even if there's more code after it.

def check_age(age): if age < 13: return "Too young" return "Old enough" # Only reaches here if age >= 13
2

RETURN VS PRINT

print() displays values but doesn't make them available to your code. return sends values back for further use.

def add1(a, b): print(a + b) # Just displays the sum def add2(a, b): return a + b # Returns sum for further use
3

MULTIPLE VALUES

Python functions can return multiple values separated by commas.

def get_dimensions(): return 10, 20, 30 length, width, height = get_dimensions() print(length)

The function returns 3 values that we can save in separate variables

4

NO RETURN

If a function doesn't have a return statement, it returns None by default.

def say_hi(): print("Hi!") # No return statement result = say_hi() print(result)

The function call will print "Hi!" but result will be None
