DECOMPOSING CODING PROBLEMS

Learning to recognize key phrases that indicate which programming concepts you need to use

1

INTRODUCTION

When reading a coding problem, certain phrases give you clues about what Python concepts you'll need to use. This worksheet will help you identify those phrases and connect them to specific Python coding techniques.

Why is this important?

Understanding how to "translate" problem descriptions into code concepts will help you:

  • Plan your solution before writing code
  • Identify what Python features you'll need
  • Make fewer mistakes in your programming
  • Solve problems more efficiently

KEY PHRASE GUIDE

Input & Output

When you see this... Think about using... Example code Notes
"Ask the user for..."
"Prompt the user to..."
"Get input from the user"		 The input() function
favorite_food = input("What is your favorite food? ") print("I like", favorite_food, "too!")
 The input function always returns a string, even if the user types a number.
"Show the user..."
"Display..."
"Output..."
"Tell the user..."		 The print() function
username = "Alex" level = 5 print("Welcome", username, "you are at level", level)
 When printing variables, do not use quotation marks around the variable name. Use commas to combine text and variables.
"Convert to a number"
"Calculate using the input"
"Do math with the input"		 Type conversion with int()
hours = int(input("How many hours do you game per week? ")) minutes = hours * 60 print("That's", minutes, "minutes!")
 Remember to convert string inputs to integers when you need to do calculations.

Mathematical Operations

When you see this... Operator Example code Notes
"Add..."
"Sum of..."
"Total of..."
"Plus..."		 Addition +
player_score = 42 bonus_points = 10 total_score = player_score + bonus_points print("Your final score:", total_score)
 When adding numbers, make sure they're converted to integers first if they came from input.
"Subtract..."
"Difference between..."
"Minus..."		 Subtraction -
total_apples = 25 eaten_apples = 7 remaining = total_apples - eaten_apples print("Apples left:", remaining)
 Be careful with the order of subtraction. The first number minus the second number is not the same as the second minus the first.
"Multiply..."
"Product of..."
"Times..."		 Multiplication *
weeks = 4 days_per_week = 7 total_days = weeks * days_per_week print("Total days:", total_days)
 The asterisk (*) symbol is used for multiplication in Python, not the × symbol.
"Divide..."
"Split..."
"Quotient of..."		 Division / or //
pizza_slices = 16 friends = 6 slices_each = pizza_slices // friends print("Everyone gets", slices_each, "slices")
 Regular division (/) gives a decimal result. Integer division (//) gives a whole number result by discarding any remainder.
"Remainder..."
"What's left over..."
"Modulo..."		 Modulo %
marbles = 17 students = 4 leftover = marbles % students print("Marbles left over:", leftover)
 The modulo operator gives the remainder after division. It's useful for checking odd/even numbers.

Conditionals

When you see this... Think about using... Example code Notes
"If..."
"When..."
"Should the..."
"Check if..."
"See if..."		 An if statement
temperature = 32 if temperature > 30: print("It's a hot day!") print("Remember to drink water.")
 Always include a colon (:) after the condition and indent the code that should run if the condition is true.
"Otherwise..."
"If not..."
"In all other cases..."		 An else statement
rain = "yes" if rain == "yes": print("Bring an umbrella!") else: print("No need for an umbrella.")
 The else block runs when the if condition is False. Remember to include a colon (:) after else too.
"If this, but if that..."
"For different cases..."
"Check multiple conditions"
"Different options based on..."		 An elif statement
score = 78 if score >= 90: print("You got an A!") elif score >= 70: print("You got a B!") else: print("You need to study more.")
 Use elif when you have multiple conditions to check. Python checks them in order, and stops at the first True condition.

Loops

When you see this... Think about using... Example code Notes
"Repeat..."
"For each..."
"A specific number of times"
"Do something for every..."		 A for loop
# Print the multiplication table for 5 for i in range(1, 11): result = 5 * i print(5, "x", i, "=", result)
 The range() function creates a sequence of numbers to iterate through. For example, range(1, 11) creates the sequence 1 through 10.
"As long as..."
"While..."
"Until..."
"Keep doing..."		 A while loop
count = 0 while count < 5: print("Count is currently:", count) count = count + 1 print("Loop has finished!")
 Be careful with while loops - if the condition never becomes False, the loop will run forever (infinite loop).

Random Values

When you see this... Think about using... Example code Notes
"Random..."
"Randomly select..."
"Choose a random..."
"Generate a random..."		 The random module
import random # Generate a random number between 1 and 20 secret_number = random.randint(1, 20) print("I'm thinking of a number between 1 and 20")
 Always import the random module at the top of your program. randint(a, b) gives a random number between a and b, including both a and b.

Special Modules

When you see this... Think about using... Example code Notes
"Draw a shape..."
"Create a graphical..."
"Turtle graphics..."		 The turtle module
import turtle import random # Create a turtle and set its characteristics t = turtle.Turtle() t.pencolor("purple") t.pensize(3) # Draw a square for i in range(4): t.forward(100) t.right(90)
 Turtle commands move a virtual "turtle" that draws as it moves. Common commands include forward(), backward(), right(), left(), penup(), pendown(), and pencolor().

IDENTIFYING CODE CONCEPTS IN PROBLEMS

Let's look at some examples from real coding problems and identify what Python concepts we need to use:

Task: Dog Years Calculator

"Write a program to ask the user their age, and then tell them their age in dog years! You can calculate a person's age in dog years by multiplying their age by 7."

Key phrases & concepts:

  • "Ask the user their age" - Getting input from the user
  • "Tell them their age in dog years" - Displaying output
  • "Multiplying their age by 7" - Using a mathematical operation
  • Working with numbers - Converting strings to integers

Task: Card Draw Challenge

"Create a simple card-based game where the player starts with a random number between 1 and 11. The player can choose whether to draw a second card. If their total exceeds 16, they lose."

Key phrases & concepts:

  • "Random number between 1 and 11" - Generating random values
  • "Player can choose whether to..." - Getting user decisions
  • "If their total exceeds 16, they lose" - Making decisions with conditions
  • "Otherwise, they win" - Alternative path for when condition is false

Task: Turtle Shape Generator

"Create a program that asks the user for the number of sides of a shape and then uses the Turtle module to draw the shape on the screen."

Key phrases & concepts:

  • "Asks the user for the number of sides" - Getting input from the user
  • "Check that the user has entered a valid number" - Validating input with conditions
  • "Set a random pen colour" - Using random values with graphics
  • "Use a for loop to draw the shape" - Repeating actions a specific number of times
  • "Display a description" - Showing output to the user

PRACTICE TIPS

When approaching a new coding problem:

  1. Highlight key phrases in the problem description that indicate what Python features you'll need
  2. List the variables you'll need to create
  3. Draw a flowchart or write pseudocode before starting to code
  4. Break down complex problems into smaller, manageable steps

Common beginner mistakes to avoid:

  • Forgetting to convert input from strings to numbers with int()
  • Using a single equals sign (=) for comparison instead of double equals (==)
  • Improper indentation in conditional statements or loops
  • Missing colons (:) after if, elif, else, for, or while statements
  • Forgetting to import modules before using them
